# Hub — управление периферией

Flask-приложение, которое работает на каждом хабе (Raspberry Pi). Обеспечивает взаимодействие с периферией (ESP8266/ESP32 и др.) через Wi-Fi.

## Что делает

- Сканирует Wi-Fi каждые 10 секунд, ищет точки доступа периферии
- Показывает список устройств с индикацией активности
- Подключает/отключает хаб к Wi-Fi точке доступа устройства через nmcli
- Скачивает логи с устройств (HTTP GET на 192.168.4.1)
- Очищает логи на устройстве
- Предоставляет REST API `/devices` для Infra Server

## Как это работает

```
ESP8266/ESP32 (AP: "KeyLog-01")          Hub (Raspberry Pi)
  192.168.4.1                      wlan0
       │                              │
       │  1. Hub сканирует Wi-Fi      │
       │◄─────────────────────────────│  iwlist wlan0 scan
       │                              │
       │  2. Оператор нажимает        │
       │     "Подключиться"           │
       │◄─────────────────────────────│  nmcli device wifi connect ...
       │                              │
       │  3. Hub запрашивает логи     │
       │◄─────────────────────────────│  GET http://192.168.4.1/
       │──────────────────────────────►│  (лог-файл)
       │                              │
       │  4. Очистка (опционально)    │
       │◄─────────────────────────────│  GET http://192.168.4.1/clear
```

Hub определяет устройства по MAC-адресу. Список известных устройств и пароли Wi-Fi хранятся в SQLite базе `Devices.db`.

## Установка

Hub автоматически устанавливается в образ при сборке (скрипт `01-build-base-pi-image-trixie.sh` копирует его в `/opt/hub` и создаёт systemd-сервис).

Для ручного запуска:

```bash
cd /opt/hub    # или путь к hub/
python3 -m venv venv
./venv/bin/pip install -r requirements.txt
python3 create_example_db.py   # создать БД (один раз)
./venv/bin/python app.py
```

## База данных

Hub использует SQLite базу `Devices.db` с двумя таблицами:

### Таблица `Devices` — список периферии

| Поле | Тип | Описание |
|------|-----|----------|
| Name | TEXT | Имя Wi-Fi точки доступа устройства (SSID) |
| Type | TEXT | Тип устройства (keylogger, badusb, deauth, ble, ...) |
| MAC  | TEXT | MAC-адрес Wi-Fi модуля устройства |

### Таблица `pass` — пароли Wi-Fi по типу

| Поле | Тип | Описание |
|------|-----|----------|
| Type | TEXT | Тип устройства (должен совпадать с Devices.Type) |
| Pass | TEXT | Пароль Wi-Fi точки доступа |

### Создание тестовой БД

```bash
python3 create_example_db.py
```

Создаст `Devices.db` с 5 тестовыми устройствами и фейковыми MAC/паролями.

### Создание БД с реальными устройствами

Для работы с настоящей периферией нужно создать базу вручную:

```bash
sqlite3 Devices.db
```

```sql
-- Создать таблицы
CREATE TABLE Devices (
    Name TEXT NOT NULL,
    Type TEXT NOT NULL,
    MAC  TEXT NOT NULL
);

CREATE TABLE pass (
    Type TEXT NOT NULL,
    Pass TEXT NOT NULL
);

-- Добавить устройства
-- Name = SSID точки доступа устройства (то, что видно при Wi-Fi сканировании)
-- Type = произвольная категория (используется для подбора пароля)
-- MAC  = MAC-адрес Wi-Fi модуля устройства (в нижнем регистре)

INSERT INTO Devices VALUES ('KeyLog-Office1', 'keylogger', 'aa:bb:cc:11:22:33');
INSERT INTO Devices VALUES ('KeyLog-Office2', 'keylogger', 'aa:bb:cc:11:22:44');
INSERT INTO Devices VALUES ('BadUSB-Reception', 'badusb', 'dd:ee:ff:11:22:55');

-- Добавить пароли Wi-Fi для каждого типа
-- Все устройства одного типа используют один пароль
INSERT INTO pass VALUES ('keylogger', 'MyKeyloggerWiFiPass');
INSERT INTO pass VALUES ('badusb', 'MyBadUSBWiFiPass');
```

**Как узнать MAC-адрес устройства:**

Прошейте микроконтроллер, включите его, и на хабе выполните:
```bash
sudo iwlist wlan0 scan | grep -B5 "KeyLog"
```
В выводе будет строка `Address: AA:BB:CC:11:22:33` — это MAC.

**Как узнать SSID:**

SSID задаётся в прошивке устройства (обычно в `WiFi.softAP("KeyLog-Office1", "password")`). Это имя, которое Hub видит при сканировании.

## API

| Endpoint | Метод | Описание |
|----------|-------|----------|
| `/` | GET | Веб-интерфейс со списком устройств |
| `/data` | GET | JSON: все устройства со статусом и временем активности |
| `/devices` | GET | JSON: список устройств (используется Infra Server) |
| `/connection_status` | GET | JSON: текущее Wi-Fi подключение |
| `/connect/<mac>` | POST | Подключиться к устройству по MAC |
| `/disconnect/<mac>` | POST | Отключиться от устройства |
| `/download_log` | POST | Скачать лог с подключённого устройства |
| `/clear_log` | POST | Очистить лог на устройстве |

## Конфигурация

Параметры в начале `app.py`:

| Параметр | Значение | Описание |
|----------|----------|----------|
| `SCAN_INTERVAL` | 10 сек | Интервал Wi-Fi сканирования |
| `ACTIVE_WINDOW` | 30 сек | Сколько устройство считается активным после последнего обнаружения |
| `DOWNLOAD_TTL` | 7 дней | Срок хранения скачанных логов (автоочистка) |

## Требования к периферии

Чтобы устройство работало с Hub, оно должно:

1. Поднимать Wi-Fi точку доступа с фиксированным SSID
2. Запускать HTTP-сервер на `192.168.4.1`
3. Отдавать данные по `GET /` (лог-файл, plain text)
4. Опционально: очищать данные по `GET /clear`

Это минимальный контракт. Реализация на стороне микроконтроллера может быть любой (ESP8266, ESP32 и др.).
